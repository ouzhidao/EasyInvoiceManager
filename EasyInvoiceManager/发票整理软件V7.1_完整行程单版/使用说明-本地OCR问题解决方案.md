# 本地OCR问题解决方案

## 问题现象

如果软件运行时出现：
```
所有OCR引擎均识别失败
或
ConvertPirAttribute2RuntimeAttribute not support
```

说明PaddleOCR本地识别在您的机器上无法正常工作。

## 解决方案（三选一）

### 方案1：降级PaddlePaddle版本（推荐先试）

1. 双击运行 `降级PaddlePaddle.bat`
2. 等待降级完成（需要联网）
3. 重新打开软件测试

**原理**：PaddlePaddle 3.3.0有兼容性问题，降级到2.6.1版本可解决。

### 方案2：使用纯API模式（最稳定）

如果方案1无效，请使用纯API模式：

1. 确保已配置百度API密钥（点击软件"配置API"按钮）
2. 双击运行 `启动纯API模式.bat`
3. 软件将只使用百度API，不再尝试本地OCR

**优点**：
- 识别精度更高
- 不受本地环境影响
- 支持所有发票类型（增值税+铁路客票）

**缺点**：
- 需要联网
- 每天有500次免费额度（超出需付费或等次日）

### 方案3：配置百度API + 自动降级

如果不想用纯API模式，可以正常启动软件，但要确保：

1. 配置百度API密钥
2. 软件会优先使用API
3. 如果API额度用完，会自动尝试本地OCR（如果本地可用）

## 百度API申请方法

1. 访问 https://ai.baidu.com/tech/ocr
2. 登录百度账号（没有就注册）
3. 点击"立即使用"
4. 创建应用，获取：
   - App ID
   - API Key
   - Secret Key
5. 在软件中点击"配置API"按钮，填入这三个值

**免费额度**：每天500次调用，对于普通使用足够。

## 验证修复

### 验证方案1（降级后）

```bash
python -c "from paddleocr import PaddleOCR; ocr = PaddleOCR(lang='ch'); print('OK')"
```

如果没有报错，说明修复成功。

### 验证方案2（纯API模式）

双击 `启动纯API模式.bat`，如果能正常启动且OCR引擎状态显示"百度API可用"，则成功。

## 常见问题

**Q: 降级后还是不行怎么办？**
A: 使用方案2（纯API模式），这是最稳定的方案。

**Q: 纯API模式每天500次不够用怎么办？**
A: 可以购买百度API的付费套餐，或等次日免费额度重置。

**Q: 没有网络怎么办？**
A: 必须修复本地OCR。尝试卸载后重新安装paddlepaddle==2.6.1，或检查系统环境。

## 技术说明

软件识别优先级：
1. 配置了API → 优先使用API
2. API失败/额度用完 → 尝试本地OCR
3. 本地OCR失败 → 报错

纯API模式：
- 完全跳过本地OCR初始化
- 只使用百度API
- 需要确保API配置正确且额度充足
